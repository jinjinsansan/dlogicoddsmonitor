# JRA オッズ急変シグナル 単独サービス化 計画書

作成: 2026-06-03 / 対象: `dlogic-odds-monitor` を独立プロダクト化

---

## 1. 概要・ビジョン

JRA の全レースのオッズをリアルタイム監視し、「**直前で不自然に動いた馬(特に急に売れた＝急落馬)**」を自動検知してユーザーに届ける単独サービス。

**コアインサイト**: 一般の馬券ファンは「オッズの**値**」は見るが、36レース分の「**変化**」を常時監視できない。そこを自動化して「賢い金の動き(歪み)」を可視化する。netkeiba は値は見せるが**変化アラートは出さない** = ここが差別化。

**ブランド**: Dlogic 本体とは分離(穴党チャンネルと同じ独立運用方針)。

---

## 2. 提供価値・差別化

- **リアルタイム急変検知**: 全JRAレースを自動監視、人間には不可能なカバレッジ
- **ストーリー性**: drop(急落)=「直前で売れた隠れ本命/関係者の動き」として訴求できる
- **実績の可視化**: シグナルの的中率・回収率を実データで提示(信頼の核)
- 競合(netkeiba等)が出していない「変化×アラート」に特化

## 3. ターゲット
- オッズの動きを気にする中〜上級の馬券ファン
- 直前情報・妙味馬を探す層
- 「賢い金」「直前の動き」に関心がある層

## 5. 検知ロジック(現状)

- **ポーリング間隔**: 発走60分前以上=20分 / 60-15分前=10分 / 15分前以内=5分。監視 08-21 JST
- **シグナル3種**:
  - **drop(急落)**: 前回比 −20%以下(お金が入った=本命化)
  - **surge(急騰)**: 前回比 +30%以上(お金が抜けた)
  - **reversal(1番人気交代)**: オッズ順1番人気の入替
- **除外**: 2倍未満 / 200倍以上 / cooldown 30分(連発防止)
- 急落−20%(敏感)/急騰+30%(鈍感)の**非対称設計**(急落=妙味が重要なため拾いやすく)

## 6. 機能(MVP → 拡張)

### MVP
- **リアルタイム急変ボード**: 全レース「今動いてる馬」を動いた順で一覧
- **シグナル実績表示**(的中率・回収率) ← トップに置く信頼の核
- レース別詳細(現オッズ + シグナル履歴)

### 拡張
- **オッズ推移グラフ**(`odds_snapshots` をタイムライン表示)
- **アラート設定**(会場/オッズ帯/シグナル種別でフィルタ通知)
- 「直前まとめ」発走5分前の最終急変ダイジェスト
- 種類別フィルタ(急落/急騰/逆転)

## 10. Phase 0 バックテスト計画(次のステップ)

**目的**: 「シグナルは勝てるのか」を作り込む前に実データで判定。効けば=サービスの核、効かなければ=閾値再調整。

**データ源**:
- `odds_snapshots`(過去オッズ履歴)→ 各レースで drop/surge/reversal を再現
- `update_race_results` 由来の結果(1着・単勝/複勝配当)で照合

**測定指標**:
- シグナル種別ごとの **的中率(単勝/複勝)・回収率・n数**
- オッズ帯別・発走何分前か別・会場別の層別
- 特に **drop シグナル馬の単勝回収率/複勝率** が最重要

**判定**:
- 回収率 100% 超 or 複勝率が母集団平均を有意に上回る → サービス化GO
- 閾値(−20%等)を振って最適点を探る

**留意**: 新VPS稼働直後で `odds_snapshots` 蓄積が浅い可能性 → 旧VPS分がSupabaseに残っていれば活用、足りなければ蓄積待ち or 過去オッズの別途取得を検討。

## 11. リスクと対策(公開前に潰す)

1. **netkeiba スクレイプ依存**(最大リスク)
   - 規模拡大で ToS/BAN・**オッズ再配信の権利問題**が顕在化(`gpts_api_plan` メモにも「netkeiba規約チェック必要」)
   - 対策: 公開前にデータ取得の合法性・安定供給を精査。規約確認。必要なら正規データソース検討
2. **Lightpanda 信頼性**(JRAオッズ取得の生命線)
   - 対策: chromium/Playwright フォールバック整備、取得失敗の監視・アラート
3. **シグナル有効性が未検証**
   - 対策: Phase 0 で先に確認(本計画の次ステップ)
4. **オッズ取得の遅延/欠損** → 「リアルタイム」を謳う以上、安定性が商品価値そのもの
