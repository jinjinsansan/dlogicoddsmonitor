- 症状: ユーザーがiPhone **Safari→PWA(ホーム画面追加)** で全レース通知ONにしたが、**自動通知が1度も届かない**。
- 切り分け済み: **手動テスト通知は3/3端末で到達済み**（＝VAPID/購読/PWA受信の経路は正常）。よって**送信ジョブ `push_sender.py` 側が送れていない**疑いが濃厚。
- 診断途中（次にやること）: VPSで
  ```
  tail -n 20 /opt/dlogic/odds-monitor/logs/push_sender.log
  cat /opt/dlogic/odds-monitor/data/sent_races.json      # 送信/スキップ記録
  cat /opt/dlogic/odds-monitor/data/races_today.json     # date と races件数(古い日付/空ならバグ)
  cd /opt/dlogic/odds-monitor && ./venv/bin/python scripts/push_sender.py   # 手動実行で挙動確認
  ```
- 想定原因の候補:
  1. **発火窓 0–5分前に「急落馬(biggest_drop)」が無い**レースは送らず`sent`に記録(=静かにスキップ)。全レースで該当しないと無通知に見える。

  3. `races_today.json` が古い日付/空 → 当日レースに一致せず送らない。
  4. post_time(JST "HH:MM")と現在時刻の窓計算ズレ。
  5. biggest_drop の race_id とスナップショットの race_id 不一致（両方12桁のはず＝要確認）。
- 関連実装: `scripts/push_sender.py`（cron `* * * * *`、odds-monitor venv、pywebpush、VAPID秘密鍵 `data/vapid_private.pem`、発火窓 `ALERT_MAX_MIN=5`、30分前基準で最大下落馬を算出）。
