**名称案(working: オッズレーダー)**

## 1. デザイン方針(ui-ux-pro-max 推奨)

| 項目 | 決定 |
|---|---|
| スタイル | **Dark Mode (OLED)** — 深い黒、高コントラスト、データ/夜間/速度感。WCAG AAA |
| フォント | **Fira Code**(数値=オッズ/変化率/馬番。等幅で桁揃え=プロのデータ感) + **Fira Sans**(本文/見出し) |
| グラフ | オッズ推移 = **Line / Area Chart**(Recharts)。現在値はパルス表示 |
| 効果 | 控えめなglow、150-300msトランジション、prefers-reduced-motion尊重 |

## 2. サイト構成(ページ / 画面遷移)

```
/                ランディング(LP) ─ ヒーローに「ライブ・ミニボード」を埋め込む
/board           ⭐ リアルタイム急変ボード(アプリ本体・全レース)
/race/[raceId]   レース詳細(オッズ推移グラフ + そのレースの急変履歴 + 出馬表)
/today           本日まとめ(当日の急変ダイジェスト)
/about           サービス説明 + 免責(情報提供であり的中保証なし)
/feedback        ベータ意見フォーム(または外部フォーム/LINE誘導)
```
遷移: LP → [今すぐ見る] → /board → レース行タップ → /race/[id]
グローバル: 上部に種別フィルタ(急落/急騰/逆転)+ 会場フィルタ、常時「LIVE」インジケータ。

## 3. ランディングページ(LP)構成

パターン: **ヒーロー=ライブ実物プレビュー(Show, don't tell)** + Bentoグリッド。

```
1. ヒーロー
   - 見出し: 「オッズは、嘘をつかない。」/ サブ: 「JRA全レースのオッズ急変をリアルタイムで。」
   - 右(PC)/下(SP): ★ライブ・ミニボード(実データが数件流れる=即・価値証明)
   - CTA: [無料で今すぐ見る](amber) + 「ベータ公開中・登録不要」
   - 上部に小さく「LIVE ● 東京/阪神/…監視中」

2. 価値訴求(3カラム/Bento)
   - ⚡ リアルタイム: 発走直前まで5分間隔で監視
   - 📊 全レース: 36レースを人が見られない、を自動で
   - 🔎 賢い金の動き: 急落=買われている馬が一目で

3. 機能紹介(Bentoグリッド)
   - 急変ボード(動いた順) / オッズ推移グラフ / 種別・会場フィルタ / 当日まとめ
   - 各カードに実スクショ or ミニ動画

4. 使い方(3ステップ)
   1) ボードを開く 2) 動いている馬を見る 3) レース詳細で推移を確認

5. 免責(誠実訴求 = 逆に信頼)
   - 「本サービスは"オッズの動き"を可視化する情報ツールです。的中・利益を保証するものではありません。」

6. CTA(再掲) + フィードバック誘導
   - [無料で使ってみる] / 「ベータ版です。感想を聞かせてください」→ /feedback or LINE
```
文言トーン: 煽らない・データドリブン・少しクール。誇大表現NG(賭け優位は否定済)。

## 5. コンポーネント指針
- SignalRow / SignalCard(種別色ボーダー+アイコン+オッズ変化)
- FilterChips(種別)・VenueSelect(会場)
- LiveIndicator(● 点滅 + 最終更新)
- OddsTrendChart(Recharts, Y軸反転=上が人気)
- SkeletonRow(読み込み中)・EmptyState(「今は急変が出ていないな」)
- アイコンは Lucide(SVG)。絵文字をUIアイコンに使わない

## 6. 実装メモ(Next.js + Vercel)
- App Router。初期データは **Server Component で fetch**(useEffect不可)。遅い部分は **Suspense でストリーミング**。
- ライブ更新: ボードは client で 30-60秒ポーリング(元データは10分間隔更新)or SWR。`revalidateTag` 併用。
- データ源: Supabase `odds_signals`(直近) / `odds_snapshots`(推移グラフ) / `race_results`(発走後)。読み取りは anon キー + RLS、または VPS API 経由(`bot.dlogicai.in` の別パス or 専用サブドメイン)。
- モバイルファースト(375/768/1024/1440)。本文16px以上、横スクロール無し。
- パフォーマンス: 画像WebP/lazy、reduced-motion対応、非同期領域は高さ予約(content-jumping防止)。

## 7. ベータ運用の注意
- **免責を明記**(情報提供・的中/利益保証なし)。誇大表現禁止(賭け優位はバックテストで否定済)。
- フィードバック導線を必ず置く(/feedback or LINE)。反応計測(滞在・再訪・フィルタ利用)。
- 「ベータ」バッジを常時表示。登録不要で即体験 → 障壁最小化。
