# dlogic 運用チェック（調査・報告のみ）

jin が **オッズくん / netkeita / dlogic** の状況・稼働・データ・文字化けについて尋ねたら、
次のコマンドを実行し、その出力をそのまま簡潔な日本語で報告する。

## 実行コマンド
```bash
python3 /root/.hermes/skills/dlogic/dlogic-ops/check_status.py
```

出力には各サービスの状態が🟢/🔴付きで並ぶ。**🔴があれば「ここが異常です」と強調**して報告する。

## 各項目の意味
- **オッズくん**: `targetLabel`=対象開催日、`mode`=preview(事前情報)/live(開催中)/finished(結果)。
  - 更新が20分以上前 → 静的JSON生成が止まっている疑い🔴
  - 開催日(土日)の日中なのに急変が0件 → Lightpandaのオッズ取得失敗の疑い🔴
- **netkeita**: その日のレース数＋文字化け検査。会場名が既知リストに無い＝**文字化け疑い**🔴。
  - 過去に「全レースの会場が空/東京になる」文字化け障害が起きたことがある(netkeibaのUTF-8移行が原因)。
- **公開サイト**: HTTP 200=正常。

## 厳守
- このスキルは **調査と報告だけ**。サーバー・データの**修正/再起動/SSH/プリフェッチ再生成などは行わない**。
- 異常を見つけたら「○○が異常です。修正が必要かもしれません(担当に連絡を)」と報告に留める。自動修復はしない(別途、確認付きで段階解放予定)。
- 補足: VPS上で15分毎の常時監視cronが動いており、異常時はTelegramへ自動通知される。本スキルは「今すぐ状況を知りたい」時の手動確認用。
